# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Commands

```bash
# Development (D1 local DB via Cloudflare runtime)
npm run dev                      # Next.js dev server at localhost:3000

# Build
npm run build                    # next build --webpack  (Turbopack is NOT used — incompatible with Cloudflare Workers)
npm run cf:build                 # OpenNext Cloudflare build (for deployment)

# Lint
npm run lint

# Database
npm run db:generate              # Generate Drizzle migration after schema change
npm run db:migrate:local         # Apply migrations to local D1
npm run db:migrate:remote        # Apply migrations to remote Cloudflare D1
npm run db:seed:local            # Seed master data (prefectures, gift_categories)
npm run db:seed-races:local      # Seed race data (seed-races.sql)
npm run db:studio                # Drizzle Studio GUI

# Cloudflare preview / deploy
npm run cf:preview               # Build + wrangler pages dev (localhost:8788)
npm run cf:deploy                # Build + wrangler pages deploy
```

> **Windows note**: Node.js commands must be run via PowerShell. Use `powershell -Command "Set-Location 'C:\Dev\krote-run'; npm run build"` when invoking from bash.

> `npm run dev` connects to local D1 via `initOpenNextCloudflareForDev()` in `next.config.ts`. Local DB is in `.wrangler/state/`.

## Architecture

### Runtime & Hosting
- **Cloudflare Pages + OpenNext** — all server-side code runs in the Cloudflare Workers runtime, not Node.js. `getCloudflareContext()` is used to access D1 bindings (`env.DB`).
- **No API routes** — data is fetched server-side in Server Components via `src/lib/data.ts`.

### i18n Routing
- **next-intl v4** with locales `['ja', 'en']`, default `ja`.
- All pages live under `src/app/[locale]/`. The root `app/layout.tsx` is a passthrough; the real HTML/body is in `src/app/[locale]/layout.tsx`.
- When `locale` comes from `params`, cast it to `Locale` type: `const locale = rawLocale as Locale`.
- Use `Link` and `redirect` from `@/i18n/navigation` (not `next/navigation`) for locale-aware navigation.
- Language switch in `Header.tsx` uses `<Link locale="en">` / `<Link locale="ja">` from next-intl.

### Data Flow
```
Cloudflare D1 (SQLite)
  ↓ Drizzle ORM (src/lib/db/schema.ts)
  ↓ getDatabase() → drizzle(env.DB) via getCloudflareContext()
  ↓ Data access functions in src/lib/data.ts
  ↓ Server Components (page.tsx files)
  ↓ Client Components (RaceList, HomeRaceSection, RaceFilter) receive data as props
```

- `getRaces()` fetches races + categories + gifts (3-query batch). Aid stations, checkpoints, access, nearby spots, weather are **only fetched in `getRaceById()`** to keep list queries fast.
- All filtering/sorting happens client-side in `filterRaces()` / `sortRacesByDate()` in `src/lib/utils.ts`.

### Styling
- **Tailwind CSS v4** — CSS-first configuration. Customisation is in `src/app/globals.css` `@theme` block, **not** `tailwind.config.ts`.
- Design tokens: `--color-primary` (#c0392b vermilion), `--color-ink`, `--color-cream`, `--color-border`, etc.
- Fonts injected as CSS variables via `next/font/google` in `[locale]/layout.tsx`: `--font-playfair` (serif headings), `--font-dm-sans` (sans UI), `--font-noto-sans-jp` (Japanese).
- Use `style={{ color: 'var(--color-ink)' }}` for design tokens in JSX rather than arbitrary Tailwind values when the token isn't mapped to a Tailwind class.

### Client vs Server Components
- Page files (`page.tsx`) are Server Components — fetch data here and pass as props.
- Interactive components (`RaceList`, `HomeRaceSection`, filter components) are Client Components (`'use client'`).
- **Leaflet / react-leaflet must be loaded with `dynamic(() => import(...), { ssr: false })`** — use the existing `CourseMapLoader.tsx` / `ElevationChartLoader.tsx` wrapper pattern.

### Race Card Variants
Two card styles exist and are toggleable on both the home page and race list:
- `RaceCard.tsx` — "Magazine" style (serif title, italic tagline, stats row)
- `RaceCardExp.tsx` — "Experience" style (description overlay on image, ✓ highlights list)

### Race Data
- Source of truth: `src/data/races/*.json` (one file per race, ~52 races for 2026).
- The JSON files are seeded into D1 via `migrations/seed-races-all.sql` (generated by `scripts/generate-seed-races.js`).
- To add a new race: create the JSON, run the seed script, apply to DB.
- `nearby_spots.type` is stored as a Japanese string (`'観光地' | '温泉' | 'グルメ' | '宿泊'`), not an enum.

### Key Type: RaceFilter
`src/lib/types.ts` defines `RaceFilter` with:
- `giftCategories: GiftCategoryId[]` — OR multi-select (array, not single value)
- Filtering logic in `filterRaces()` in `src/lib/utils.ts`
